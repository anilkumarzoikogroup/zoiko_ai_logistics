# Legacy migrations archive

These 43 files (`0001_p0_all_25_tables.py` through `0043_claim_lines.py`) are the
original incremental migration history. They are **not** read by alembic —
they don't live in `versions/`, so they have no effect on `alembic upgrade`,
`alembic current`, or any other alembic command.

They're kept here purely for historical/audit reference. The live database's
`alembic_version` was stamped to the new consolidated chain on 2026-06-24:

```
0001_unified_baseline.py       (versions/, was: 0001-0040 + shared parts of 0036/0041)
0002_integration_tables.py     (versions/, was: 0019/0041/0042/0043's slice-specific parts)
```

The new baseline was generated by introspecting the live database's actual
schema (not by replaying these old files — several of them, like 0032-0035,
correct drift introduced by earlier ones, so the live DB's real structure was
the source of truth) and validated by:
1. Executing it standalone against a throwaway database and diffing table/
   column/constraint/index counts against the live DB — exact match.
2. Running it through real `alembic upgrade head` against a second throwaway
   database — exact match again.
3. Confirming `alembic upgrade head` against the live DB is a no-op after the
   stamp (already at head, nothing pending).

See `../versions/SLICE_MAP.md` for which parts of the schema are shared platform
spine vs. genuinely slice-specific, and `backend/slices/*/SCHEMA.md` for each
slice's own ownership summary.
